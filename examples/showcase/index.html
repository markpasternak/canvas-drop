canvas-drop

Static files, with backend superpowers.

This whole page is a folder of static files deployed to a URL — no server code, no build step. It gains a backend only through five small primitives, called from the browser. Everything below is live.

How it works

  1. Drop a folder (HTML/CSS/JS) onto canvas-drop. It serves at a URL.
  2. Add one script tag — then call canvasdrop.* from your code:
<script src="/sdk/v1.js"></script>

const me = await canvasdrop.me();              // who's viewing
await canvasdrop.kv.increment("views");        // shared state
await canvasdrop.files.upload(file);           // file storage
canvasdrop.ai.stream(messages, { model });     // streamed AI
canvasdrop.realtime.channel("room").publish(); // live pub/sub
…

me() Identity

Every request already carries the viewer's verified org identity (from the login proxy / SSO). canvasdrop.me() reads it server-side — your canvas never has to build auth.

Show the code 
const me = await canvasdrop.me();
// { id, email, name, avatarUrl }
greeting.textContent = `Hello, ${me.name}`;

kv Key–value storage

A tiny persistent store, two namespaces: shared (everyone sees it) and per-user (private to each viewer). Great for counters, settings, and small app state.

·shared page views
Show the code 
// shared, atomic — returns the new total
const views = await canvasdrop.kv.increment("views");

// per-user namespace, isolated per viewer
await canvasdrop.kv.user.set("note", text);
const mine = await canvasdrop.kv.user.get("note");

files File storage

Upload, list, and delete files (images, data, anything). Each gets a stable URL you can drop straight into an <img> or link.

    Show the code 
    const f = input.files[0];
const { id, url } = await canvasdrop.files.upload(f);
img.src = url;                       // or canvasdrop.files.url(id)
const all = await canvasdrop.files.list();
await canvasdrop.files.delete(id);

    ai AI chat (streamed)

    A server-side LLM proxy. The provider key stays on the server — never in this page. Responses stream token-by-token; usage and cost come back when it's done.

    Show the code 
    const messages = [{ role: "user", content: prompt }];
for await (const delta of canvasdrop.ai.stream(messages, { model })) {
  bubble.textContent += delta;       // render as it streams
}

    realtime Live presence & pub/sub

    Ephemeral WebSocket channels per canvas: see who's here, broadcast messages, run a live poll. Nothing is persisted — durable state goes in KV.

    Tip: open this page in a second tab to watch broadcasts and the poll sync live. (Presence lists distinct people — two tabs signed in as the same user count once.)
